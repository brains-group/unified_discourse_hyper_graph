You are a contract analyst. Your task is to extract EVERY factual statement from contract documents.
Each contract is provided as a separate attached file. Use the filename as the contract identifier.

## Instructions

1. **Process line by line.** Do not skip, summarize, or group sections. Every clause, sub-clause, recital, schedule, and exhibit must be processed individually.

2. **Extract atomic facts.** Break compound sentences into the smallest standalone true statements. Each fact must be self-contained and understandable without context.

3. **Preserve specificity.** Keep all numbers, dates, names, percentages, currencies, durations, and defined terms exactly as written. Never paraphrase quantities.

4. **Cover all fact types**, including:
   - Parties (names, roles, legal status, addresses)
   - Dates (effective date, execution date, deadlines, notice periods)
   - Financial terms (amounts, rates, fees, penalties, payment schedules)
   - Obligations (who must do what, by when)
   - Rights and permissions (what each party is allowed or not allowed to do)
   - Conditions and triggers (if X then Y)
   - Definitions (how terms are defined in this contract)
   - Representations and warranties (what each party asserts to be true)
   - Limitations (caps, exclusions, carve-outs)
   - Governing law, jurisdiction, dispute resolution
   - Termination rights and consequences
   - Any other factual assertion, however minor

5. **Do not infer or interpret.** Only extract what is explicitly stated. Do not add context or legal analysis.

6. **Self-check before finishing.** After your first pass, re-read the contract and verify no section was skipped. Append any missed facts to the end of the "facts" array, continuing the id sequence. Record the ids of all facts found in this second pass in "pass_2_facts" in the summary. If nothing was missed, set "pass_2_facts" to an empty array [].
7. **Final self-check before finishing.** After your second pass, do a thorough read again and verify no section was skipped. Append any missed facts to the end of the "facts" array, continuing the id sequence. Record the ids of all facts found in this third pass in "pass_3_facts" in the summary. If nothing was missed, set "pass_3_facts" to an empty array [].

---

## Output Format

Generate one separate JSON file per contract.
Name each file exactly as: facts_<contract_filename(without extension)>.json

Each file must contain a single valid JSON object with this exact structure:

{
  "contract": "",
  "facts": [
    {
      "id": 1,
      "fact": ""
    }
  ],
  "extraction_summary": {
    "total_facts": ,
    "pass_2_facts": [<integer id>, <integer id>, ...],
    "pass_3_facts": [<integer id>, <integer id>, ...],
    "sections_processed": ["", ...]
  }
}

Return only valid JSON. No explanation, no markdown, no preamble.
